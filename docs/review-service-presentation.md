리뷰 서비스 발표 자료 (Draft)
==========================

목표/배경
--------
- 리뷰는 구매(대여) 경험의 신뢰 지표로, 구매 전환에 직접 영향을 줌
- 리뷰 데이터는 많은 읽기 트래픽이 예상되어 조회 최적화가 중요
- 리뷰 요약(한줄 요약) 기능은 AI 처리 비용/지연을 고려해 비동기 파이프라인 설계

서비스 범위
----------
- 리뷰 CRUD (작성/수정/삭제/조회)
- 판매자 리뷰 요약(요약 텍스트 + 집계 지표)
- 리뷰 요약 버킷 관리 및 최종 요약 생성 트리거

도메인 모델
-----------
- Review: 리뷰 본문, 평점, 작성자, 판매자, 상태(soft delete)
- ReviewSummary: 최종 요약, 총 리뷰 수, 평점 합
- ReviewSummaryBucket: 일정 수의 리뷰를 묶은 버킷 요약

핵심 설계 의도
-------------
- 리뷰 조회는 자주, 요약 생성은 비싸고 느림
- 그래서 "버킷 요약"으로 중간 결과를 누적하고, 최종 요약은 최근 리뷰 범위 기준으로 합성
- 리뷰 작성/수정/삭제 시 집계 지표를 바로 업데이트해 조회 부하 최소화

API 개요
--------
- POST /api/reviews: 리뷰 작성
- PATCH /api/reviews/{reviewId}: 리뷰 수정(작성자만)
- DELETE /api/reviews/{reviewId}: 리뷰 삭제(soft delete, 작성자만)
- GET /api/reviews?sellerId=...: 판매자 리뷰 목록(필터/정렬/페이징)
- GET /api/reviews/me: 내 리뷰 목록(필터/정렬/페이징)
- GET /api/reviews/summary?sellerId=...: 판매자 리뷰 요약(없으면 204)

리뷰 작성 흐름
-------------
1) 대여 반납 완료 여부, 작성 가능 기간, 중복 작성 여부 검증
2) Review 저장
3) ReviewSummary 집계(총 리뷰 수, 평점 합) 갱신
4) 요약 생성 조건 충족 시 요약 요청 이벤트(Outbox) 발행

리뷰 수정/삭제 흐름
------------------
- 작성자 본인만 수정/삭제 가능
- 수정 시 평점 변경분만 집계에 반영
- 삭제는 soft delete, 집계에서 리뷰 수/평점 합 차감

요약 생성 전략
-------------
- a(minTotalCount): 최초 버킷을 만들 최소 리뷰 수
- b(minNewCount): b개 리뷰가 누적될 때마다 버킷 요약 생성
- c(recentLimit): 최종 요약은 최근 c개 리뷰를 커버하는 버킷들을 합성
- maxLength: 요약 최대 길이

요약 생성 파이프라인
--------------------
- ReviewService -> ReviewSummaryService -> ReviewOutbox(요청 적재)
- Kafka로 AI 요약 요청 이벤트 발행
- ai-service가 요약 생성 후 결과 이벤트 발행
- review-service가 결과 수신 후 DB에 저장

최종 요약 생성 시점
------------------
- 리뷰 작성으로 새 버킷이 생성되고, 최종 요약이 비어있는 경우 즉시 생성
- 스케줄러(크론)로 주기적인 최종 요약 갱신

데이터 일관성/성능 고려
----------------------
- 리뷰 목록은 soft delete 제외 조회
- 집계 값은 리뷰 변경 시점에 즉시 업데이트(읽기 최적화)
- 요약 생성은 비동기로 처리하여 사용자 요청 지연 최소화
- 버킷 방식으로 재요약 범위를 줄여 비용과 지연을 제어

실패/예외 처리
-------------
- 대여 정보 미확인/불일치/미반납/기간 경과 시 작성 금지
- 중복 리뷰 작성 방지
- 요약 결과 이벤트가 비정상일 경우 저장하지 않음

장점
----
- 리뷰 조회 성능 최적화
- 요약 생성 비용을 버킷 기반으로 안정화
- 비동기 처리로 사용자 경험 유지

한계/보완 여지
-------------
- 요약 생성 지연으로 실시간성과는 트레이드오프
- 이벤트 파이프라인 장애 시 요약 갱신 지연 가능
- 권한 설정 누락 시 요약 조회 접근성 이슈 가능

발표용 핵심 한줄 요약
---------------------
"리뷰는 즉시 저장·집계하고, 요약은 버킷 기반 비동기 파이프라인으로 비용/지연을 제어하는 구조"
